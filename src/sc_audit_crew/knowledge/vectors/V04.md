# V04 — MEV

## What to check
1. Swap/deposit functions — is there slippage protection (minAmountOut) and a deadline parameter? Absence enables sandwich attacks.
2. Liquidation functions — can a liquidation be frontrun or backrun to extract value beyond the intended liquidation bonus?
3. Admin operations — fee changes, rate updates, parameter changes that users can observe in the mempool and frontrun.
4. Functions where tx ordering changes economic outcome — identify explicitly and describe the ordering dependency and its impact.

## Admin frontrunning note
Even contracts without swap functions expose admin op frontrunning risk (e.g., users rushing to act before a fee increase applies). This is typically Low or Medium unless direct fund extraction is possible.

## Severity guidance
- No slippage protection on user-facing swap or deposit: High.
- Liquidation mechanism exploitable through frontrunning for material value: High or Medium.
- Admin op frontrunning with direct fund impact: Medium.
- Admin op frontrunning with no material financial consequence: Low.
- Ordering dependency with no economic impact: Info.
