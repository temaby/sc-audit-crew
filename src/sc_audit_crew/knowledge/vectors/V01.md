# V01 — Reentrancy

Apply this three-tier model to every external call identified in the FUNCTION INVENTORY.

## Tier 1 — Same-contract re-entry

A re-entrant call into the same or another mutating function on THIS contract.

- `nonReentrant` modifier present on the function: Tier 1 re-entry is fully blocked **for all other `nonReentrant`-protected functions**. Emit as **Informational** with a defense-in-depth note if CEI is also violated; do NOT emit at all if CEI is correctly ordered.
  - **Unguarded sibling functions:** `nonReentrant` only blocks re-entry into other guarded functions. If the contract exposes other state-mutating functions that lack `nonReentrant`, those remain callable during the outbound call window. Check whether any such unguarded function is exploitable via the re-entry path — if yes, emit a Tier 1 finding at the appropriate severity as if the guard were absent.
  - **Upgradeable proxy context:** If the contract uses a proxy pattern, verify the `ReentrancyGuardUpgradeable` variant is used, not the non-upgradeable `ReentrancyGuard`. The non-upgradeable variant stores `_status` at sequential slot 0, which in an upgradeable implementation collides with the first state variable. This is a storage-collision defect — flag it under V06 (item 1), not here.
- Without `nonReentrant`: if state guarding re-entry is written AFTER the external call, this is a CEI violation. Severity depends on callback path (see below).

## Tier 2 — Read-only stale value (same contract)

A view on THIS contract is called externally while an outbound call is in flight; stale value has no on-chain financial consequence.

- Maximum severity: Low.
- `nonReentrant` does NOT block this tier.

## Tier 3 — Cross-contract stale state

A DIFFERENT in-scope contract reads this contract's state while its outbound call is in flight AND uses that stale value for a financial decision (price, collateral, liquidation).

- `nonReentrant` on THIS contract does NOT block Tier 3.
- Verify ALL THREE: (a) another contract calls a view here, (b) the result drives a financial decision, (c) the value can be stale during the outbound call.
- All three YES → High or Critical.

## Severity guidance

- Tier 1 + ERC777 or custom callback token, no nonReentrant: High or Critical depending on confirmed fund-loss path.
- Tier 1 + standard SafeERC20 only (no callback path), no nonReentrant: cap at High — unconditional exploit requires a hookable token as precondition.
- Tier 1 with nonReentrant present: CEI violation is defense-in-depth only; Informational at most.
- Tier 2: Low at most.
- Tier 3 with confirmed financial-loss path: High or Critical.

## Note for peer reviewer

Findings citing ONLY a CEI pattern violation (no Tier 2 or Tier 3 path identified) are subject to the REENTRANCY-CEI-ONLY gate in the calibration task.
