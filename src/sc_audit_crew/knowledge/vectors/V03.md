# V03 — Oracle Manipulation

## What to check
1. Chainlink latestRoundData() — all four return values must be validated:
   - `answeredInRound >= roundId` — detects stale data
   - `updatedAt + staleness_threshold >= block.timestamp` — staleness window
   - `answer > 0` — sanity check (negative price is invalid)
   - `roundId != 0` — unstarted round guard
2. Single-block spot price (Uniswap slot0, observe with tiny window) — manipulable within one transaction.
3. TWAP window adequacy — minimum 30 minutes for most protocols; shorter windows are manipulation vectors.
4. Price feed decimal assumptions — verify the assumed decimal count matches the actual feed (Chainlink ETH/USD = 8 decimals, not 18).
5. L2 sequencer uptime — on Arbitrum/Optimism, is the sequencer uptime feed checked before consuming Chainlink data?

## Severity guidance
- Missing staleness check on price feed driving collateral or liquidation decisions: High.
- Single-block spot price used for any financial calculation: High.
- TWAP window too short to resist manipulation with realistic capital: Medium.
- Missing sequencer uptime check on L2: Medium.
- Informational validation gap with no direct exploit path: Low or Info.
