# V05 — Flash Loan Composability

## What to check
1. Price-sensitive computations manipulable within one transaction using borrowed liquidity.
2. State changes that assume token balances cannot change between function entry and exit.
3. Re-entry into a price-sensitive function after borrowing flash liquidity within the same transaction.
4. The general pattern: borrow → manipulate state → profit → repay → all in one transaction.

## Severity guidance
- Flash loan enables direct extraction of user or protocol funds: Critical.
- Flash loan enables price manipulation leading to undercollateralised borrowing or incorrect liquidation: High or Critical.
- Flash loan enables governance vote manipulation: High.
- Flash loan composability risk with no concrete exploit path identified: Medium.
