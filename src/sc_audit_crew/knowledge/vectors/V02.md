# V02 — Access Control

## What to check
1. Every public/external function — does a modifier or `require` restrict the caller to authorised roles?
2. Functions that should be restricted but are not: minting, fee changes, admin fund withdrawal, parameter setters.
3. Privilege escalation paths — can any actor become owner or admin through a call sequence not intended by the design?
4. Low-level return values — every `call()`, `delegatecall()`, `staticcall()` return value must be checked; unchecked false return is a silent failure.

## BOUNDARY
V02 covers ONLY low-level call()/delegatecall()/staticcall() return values.
ERC20 token.transfer()/transferFrom() return value checks belong to V14.

## Severity guidance
- Unrestricted function allowing direct fund extraction: Critical or High.
- Unrestricted setter enabling protocol misconfiguration with indirect fund impact: High or Medium.
- Unchecked low-level call return value where false return loses value: High.
- Missing access control on informational or view-only function: Low or Info.
