# V09 — Signature Replay

## What to check
1. Nonce — is a per-signer nonce incremented atomically with each signature use? (Mapping from signer → nonce, incremented before the operation completes.)
2. Domain separator — must include: contract address, chain ID, contract name/version. Missing any of these enables cross-contract or cross-chain replay.
3. Chain ID — if missing, a valid signature on mainnet replays identically on any EVM chain sharing the same contract address.
4. Cross-contract replay — can a signature for contract A be submitted to contract B if they share the same interface and domain separator structure?
5. EIP-712 compliance — is `hashTypedData` (EIP-712) used, or raw `keccak256(abi.encodePacked(...))` (vulnerable to type collision)?
6. Permit functions (EIP-2612) — validate: deadline not expired, signer recovered correctly, nonce incremented, owner != address(0).

## Severity guidance
- Missing nonce enabling unlimited replay draining funds: Critical.
- Missing chain ID enabling cross-chain replay with direct fund loss: High.
- Missing domain separator (contract address) enabling cross-contract replay: High.
- Permit implementation frontrunnable to grief but not steal: Low or Medium.
