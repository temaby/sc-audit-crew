# V13 — Asset Accounting

## LEDGER COMPLETENESS CHECK

Every path where tokens enter or leave the contract must update the corresponding internal ledger variable.

Verify: for every deposit path, internal balance increases by the correct amount.
Verify: for every withdrawal path, internal balance decreases by the correct amount.
Verify: internal balance == actual token balance at all exit points.

## HISTORICAL VIEW FUNCTION AUDIT (PT-C)

Run this loop for EVERY external or public view function that accepts a timestamp or block number as a parameter:

1. List all such functions explicitly by name.
2. For each: verify the return value is computed purely from state AS OF that timestamp — not from any post-timestamp accumulated state.
3. Anti-pattern to detect: `return max(f(timestamp), currentTotal)` where `currentTotal` includes activity after `timestamp`. This corrupts governance snapshots, reward calculations, and off-chain auditing tools — even when individual withdrawals are mathematically correct.
4. Emit MEDIUM for every such function found — this is an externally observable integrity violation, not a gas concern.

**BOUNDARY-VALUE SWEEP — multi-phase conditional logic:**
For any view function whose result depends on multiple timestamp comparisons (e.g. before-cliff / cliff / linear / fully-vested phases):

1. List every branch condition and the phase it represents.
2. Identify the full set of phase transition points (cliff timestamp, start timestamp, end timestamp, any others).
3. For each transition point T, evaluate the function with `_referenceTs` at: T − 1, T, T + 1.
4. Verify the returned value is consistent with the correct phase for each of those three inputs. An off-by-one or wrong comparison operator (`>=` vs `>`) can cause the wrong phase to fire at the boundary.
5. Also construct a scenario where two phase conditions are satisfied simultaneously (e.g. `cliffReleaseTimestamp <= _referenceTs` AND `_referenceTs < startTimestamp`). Verify the returned value is intentional for that overlap.
6. If any boundary input returns an unexpected value: emit MEDIUM.

## Severity guidance

- Internal ledger not updated on token exit, creating insolvency over time: High or Critical.
- Fee path that accumulates tokens without tracking in ledger: High or Medium.
- Historical view function returning post-timestamp accumulated state: Medium.
- Minor ledger rounding discrepancy with no insolvency risk: Low or Info.
