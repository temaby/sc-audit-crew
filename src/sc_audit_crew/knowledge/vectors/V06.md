# V06 — Proxy / Delegatecall

## What to check

1. Storage slot collisions — does the proxy layout reserve slots per EIP-1967? Verify implementation and admin slots.
2. Uninitialized implementation — **do not check `_disableInitializers()` here; that sub-check is owned by V17.**
   Only flag issues specific to proxy-level initializer exposure not already captured by V17 (rare).
3. `selfdestruct` in implementation — if reachable, renders the proxy permanently inoperable.
4. `msg.sender` context — functions in the implementation using `msg.sender` behave differently when called directly vs via proxy.
5. Upgrade governance — who controls `upgradeTo`? Single EOA, multisig, timelock, or governance?

## Severity guidance

- Storage collision causing state corruption: High or Critical.
- `selfdestruct` in implementation reachable by non-owner: High.
- Upgrade controlled by single EOA with no timelock: Medium (admin malice precondition applies — MANDATORY DOWNGRADE RULE in calibration).
- `msg.sender` confusion documented but not exploitable in practice: Low.
